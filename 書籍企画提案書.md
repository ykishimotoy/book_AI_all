## 2. この本は何か（ひとことで）

XやZennで生成AIの断片情報しか追えていない人が、15分野の「勢力図」と「変わらない選定軸」を一冊で手に入れ、どのAIツールを・いつ・いくらで・何に注意して使えばいいかを自分で判断できるようになる本。

## 3. 対象読者

### メインターゲット
- ZennやXでAI情報を日常的に追っている初心者〜中級者（20〜40代のITパーソン・個人開発者・クリエイター）
- ペイン：情報が断片的すぎて全体像がつかめない。「Seedanceがすごいらしい」「Copilotが改悪されたらしい」という点の情報は入るが、線と面（勢力図・選び方・費用感）が見えない。SNSの情報は玉石混淆で、料金・商用利用条件・日本語対応といった「実際に使うときに必要な情報」が抜けている
- 現在の知識レベル：ChatGPTやClaudeは日常的に使う。画像・動画・音楽などは「触ったことがある」程度。API・ローカルLLMは名前を知っているレベル

### サブターゲット
- IT企業・事業会社でAI推進役を担う担当者（情シス・DX推進・企画部門）
- 求めているもの：業務が忙しく最新情報を追い切れない中で、社内への説明・ツール選定・稟議に使える「信頼できる俯瞰情報」。特に商用利用条件・データの取り扱い・中国系サービスのガバナンス観点など、社内で必ず聞かれるポイントの整理

### なぜこの読者に届くのか
既存のAI書籍は「ChatGPT活用術」「Claude Code入門」といった単一ツールの深掘り本か、生成AIの概念解説本に二極化しており、**「全分野を横断して勢力図と選び方を示す本」が市場に存在しない**。ネット情報は速いが断片的・広告混じり・信頼性不明で、体系性を求める読者の不満は大きい。本書は「網羅性 × 選定軸 × 費用と注意点のリアル」を統合して提供し、その空白を埋める。

## 4. 企画概要・コンセプト

生成AIは2026年、「ChatGPTだけ知っていればいい時代」を完全に終えた。基盤モデルはOpenAI・Anthropic・Googleの三つ巴に加えローカルモデルが実用期に入り、動画では中国勢（ByteDance・Kuaishou）が首位を争い、日本語音声では個人開発のOSSが商用サービスを脅かす。分野ごとに勝者が違い、勢力図は数ヶ月で入れ替わる。

この状況で読者に必要なのは「今週の最新情報」ではなく、**「全体の地図」と「変わらない羅針盤」**である。本書は15分野（基盤モデル〈クラウド／ローカル〉、コード生成、コーディング・作業エージェント、スライド、画像、動画、音楽、ボイス、効果音、リサーチ、議事録・音声認識、翻訳、3D、検索AI）について、①勢力図、②代表ツールと料金の目安、③日本語対応と日本からの利用可否、④商用利用・著作権・データ主権の注意点、⑤選定フローチャート、を統一フォーマットで提示する。

さらに、個別ツール情報（変わる情報）と選定軸・注意点の構造（変わらない情報）を意図的に分離して執筆することで、陳腐化に強い構造をとる。そして変わる情報は、著者が運営する無料AI学習サイト「すごいAI道場」内の本書サポートページで継続更新する。**「紙で骨格を、Webで鮮度を」**が本書の設計思想である。

**この本ならではの独自性：**
- **15分野を統一フォーマットで横断する網羅性**：単一ツール本・概念本ばかりの市場で唯一の「全地図」ポジション
- **「使うときに困る情報」への徹底フォーカス**：料金の実質相場、無料プランの商用利用NG、日本語ボイスの崩れやすさ、中国系サービスのデータガバナンス、画像・動画出力の文化的バイアス（米中寄りになりがち）など、SNSでは流れてこない実務情報を明記
- **サポートサイトによる継続更新**：AIツール本最大の弱点「陳腐化」を、著者運営サイトとの連携で構造的に解決。書籍単体で完結させない設計は類書にない
- **著者自身が「全部払って全部使っている」当事者性**：Kling・Runway・Suno・Claude Code等を実際にサブスク契約し、AI映画祭運営・楽曲制作・エージェント開発まで行う著者の一次体験に基づく費用感とハマりどころ

**読者が得られるもの（ベネフィット）：**
- 15分野それぞれの勢力図が頭に入り、新ツールのニュースを「地図上のどこの話か」で理解できるようになる
- 用途・予算・リスク許容度から、自分に合ったツールを自力で選定できるようになる
- 商用利用・著作権・データ取り扱いの地雷を踏まずにAIを業務・創作に導入できる

## 5. なぜ売れるか・なぜ今か

- **市場の追い風／時流**：生成AI関連書籍市場は拡大が続き、Amazonの新着・ベストセラーはAI書が席巻。2026年は動画（Seedance 2.0/2.5・Kling 3.0）、画像（GPT Image 2・Nano Banana 2）、コーディングCLI競争、ローカルLLM実用化など分野別の地殻変動が同時多発。さらにSoraの全面撤退やGemini CLIの終了といった「大手の看板サービスが突然消える」事件が相次ぎ、断片情報ではなく「全体地図と選び方の軸」を求める読者の切実さがかつてなく高まっている
- **市場の空白**：類書は「ChatGPT本」「Claude Code本」「Suno本」など単一ツール特化が主流。複数分野を扱う本も「初心者向けの浅い紹介」にとどまり、勢力図・料金相場・商用利用リスクまで踏み込んだ横断本は存在しない
- **差別化**：①横断性と深さの両立、②サポートサイト連携による鮮度維持、③地方発・コミュニティ発の著者ストーリー（札幌200人コミュニティ、北海道初のAI映画祭）による広報フックの強さ

## 8. 構成案（目次）

### はじめに ─ タイムラインを閉じて、地図を開こう
毎日流れてくるAIニュースに焦りながら、全体像がつかめない読者への共感 → 「変わる情報」と「変わらない軸」を分ければAIは追える → 本書の使い方（地図として引く／羅針盤として読む）とサポートサイトの案内。

---

### 第1部　地図の読み方 ─ 全分野に共通する原則
> 個別ツールを見る前に、どの分野にも通用する「変わらない判断軸」を先に手に入れる。ここが本書の陳腐化しない骨格。

- **第1章　2026年、生成AIの勢力図はこうなっている** … 米OpenAI/Anthropic/Google三強＋中国勢＋OSSという構図の俯瞰。「分野ごとに勝者が違う」時代の全体マップ。読者はニュースを地図上に置けるようになる
- **第2章　AIツール選定の羅針盤** … 用途・予算・リスクの3軸選定法。無料枠と商用利用の罠、サブスクとAPI従量課金の使い分け、ドル建て料金の読み方（$1=160円前後・2026年7月時点基準）。読者は「自分で選ぶ」型を得る
- **第3章　全ツール共通の地雷原** … ハルシネーション、著作権（学習データ訴訟・既存IPの類似生成・無料プラン商用NG）、データ主権（中国系サービスと社内ポリシー）、出力の文化的バイアス（画像・動画が米中寄りになる問題）、日本語の壁。読者は導入前チェックリストを持ち帰る

### 第2部　15分野の全地図 ─ 勢力図・料金・注意点
> 本書の中核。各章とも「勢力図 → 代表ツールと料金目安 → 日本語対応 → 注意点 → 選定フロー」の統一フォーマット。

- **第4章　基盤モデル（クラウド）** … GPT・Claude・Geminiの三つ巴と選び分け。チャットUI・サブスク・APIの3層で見る料金構造
- **第5章　基盤モデル（ローカル）** … Gemma 4・gpt-oss・Qwen・DeepSeek等の実用期。Ollama/LM Studioでの始め方、クラウドとの損益分岐、機密データという最強のユースケース
- **第6章　コード生成とコーディングエージェント** … Claude Code・Codex・Antigravity（旧Gemini CLI）・Copilot・Cursor・Kiro等。CLI型 vs IDE型、そして2026年上半期の激震（Gemini CLI終了、Copilotの新規停止・従量課金化）から学ぶ「梯子を外されない」選び方
- **第7章　作業エージェント** … Devin・Manus・Genspark、そしてコーディングエージェントの汎用化。「委任する働き方」の現在地
- **第8章　スライド・資料生成** … NotebookLM・Gamma・Canva・Genspark・Claude系。編集可能性（PPTX出力の質）と図の自動生成という2大評価軸
- **第9章　画像生成** … GPT Image 2 vs Nano Banana 2の2強と、Midjourney等の専門勢。日本語テキスト描画の進化、思考する画像生成
- **第10章　動画生成** … Seedance 2.0/2.5・Kling 3.0・Veo 3.1の勢力図と、Sora全面撤退（2026年）が象徴する市場再編。アニメ・セル調表現の実力、プラットフォーム経由利用（Runway等）という潮流、個人クリエイターのコスト設計
- **第11章　音楽生成** … Suno一強とUdio・Lyria。商用利用の権利構造（所有権ではなく利用権）、係争中の訴訟リスク
- **第12章　ボイス・効果音** … 日本語特化OSS「Irodori-TTS」の衝撃とElevenLabs。日本語ボイス特有の難しさ（読み・イントネーション）と対策
- **第13章　リサーチ・検索・議事録・翻訳** … Deep Research系・Perplexity・NotebookLM・Whisper系議事録・AI翻訳。ホワイトカラー業務の定番構成
- **第14章　その他の注目分野** … 3D生成、オンデバイスAI、ブラウザエージェント等、次に来る分野の早見

### 第3部　地図を使いこなす ─ 実践と持続
> ツールを組み合わせ、変化に追いつき続けるための実践編。

- **第15章　組み合わせて使う：目的別ワークフロー** … 「1本のAI映像を作る」「登壇資料を1時間で作る」「社内AI導入を通す」等、複数分野をまたぐ実践レシピ。著者のAI映画祭・映像制作の実体験ベース
- **第16章　月いくら払うか：個人と組織のAI予算設計** … 個人クリエイター月5千円/2万円/5万円の3モデルケース、企業のスモールスタート構成。サブスク断捨離の考え方
- **第17章　地図の更新し続け方** … 情報収集の型（一次情報の見分け方、コミュニティ活用）、そして本書サポートページ（すごいAI道場内）での勢力図アップデートの使い方
